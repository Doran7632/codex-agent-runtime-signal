# codex-agent-runtime-signal v1.4.0

## English

### Highlights

- Renames the project consistently to `codex-agent-runtime-signal`, with Swift targets under `CodexAgentRuntimeSignal` and environment variables under `CODEX_AGENT_RUNTIME_SIGNAL_*`.
- Shows only currently open Codex sessions in the menu bar, one row per session, with host app, running/idle status, and the Codex resume session name.
- Adds independent task completion and permission-blocked bubbles below the MacBook notch, with separate sound settings, stacking behavior, and blue/red particle effects.
- Removes the floating window and the legacy Run settings tab to keep the app smaller and lower overhead.
- Reduces resident CPU usage by trimming polling frequency, avoiding redundant display recomputation, caching display metadata, coalescing repeated active heartbeats, and lowering animation cost.

### Changes

- Host application detection now covers terminal, IDEA, VS Code, Xcode, Codex Desktop, Obsidian ACP, OpenDesign, and other apps that expose a traceable Codex process/session file.
- Idle open sessions stay visible without falling back to stale historical `codex resume` entries.
- Task bubbles use a compact dark-gray capsule style with fixed width, scrolling long session names, and independent sound selection.
- Release scripts, workflows, docs, CLI wrappers, hook scripts, source directories, and tests now use the corrected `signal` spelling.

### Fixes

- Fixes unstable ordering for multiple idle sessions from the same host app.
- Fixes idle sessions briefly disappearing when a single process scan misses an already confirmed open Codex session.
- Fixes completion bubbles firing from old completed events after a session has already started running again.
- Fixes the previous typo in the project slug; the corrected spelling is now used throughout the project.

## 简体中文

### 亮点

- 项目命名统一为 `codex-agent-runtime-signal`，Swift target 使用 `CodexAgentRuntimeSignal`，环境变量使用 `CODEX_AGENT_RUNTIME_SIGNAL_*`。
- 菜单栏只展示当前打开的 Codex 会话，一行一个 session，显示宿主应用、运行中/空闲状态和 Codex resume 会话名称。
- 新增独立的任务完成/权限卡住气泡，固定在 MacBook 刘海下方，支持独立音效、堆叠显示和蓝/红粒子效果。
- 删除悬浮窗和旧的“运行”设置页，让软件更小、更轻。
- 通过降低轮询频率、减少重复展示计算、缓存展示元数据、合并重复 active 心跳、降低动画成本来减少常驻 CPU 占用。

### 改进

- 宿主应用识别覆盖桌面终端、IDEA、VS Code、Xcode、Codex Desktop、Obsidian ACP、OpenDesign，以及其他能暴露 Codex 进程或 session 文件的应用。
- 当前打开但空闲的会话会稳定显示，不再回退展示已关闭的历史 `codex resume` 会话。
- 任务气泡改为紧凑深灰胶囊样式，固定宽度，长会话名可滚动，音效可单独选择。
- 发布脚本、GitHub Actions、文档、CLI wrapper、hook 脚本、源码目录和测试目录都已改成正确的 `signal` 拼写。

### 修复

- 修复同一宿主应用下多个空闲会话排序跳动。
- 修复单次进程扫描漏报时，已确认打开的空闲会话短暂消失。
- 修复会话重新运行后，旧完成事件误触发完成气泡。
- 修复项目 slug 的旧拼写错误；当前项目已统一使用正确拼写。

- `bash -n script/*.sh scripts/*`
